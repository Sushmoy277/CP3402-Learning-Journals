## Content Insights

This week’s practical focused on creating and working with WordPress child themes. Through this task, I developed a clearer understanding of how themes function in WordPress and how developers safely customise websites without affecting the original theme files.

One of the key things I learned is that directly editing a parent theme is not a good practice. If the theme is updated, all changes can be overwritten. A child theme solves this issue by allowing customisations to be stored separately, while still inheriting all features from the parent theme. This approach makes websites more stable and easier to manage.

While completing the practical, I created a new child theme folder and added the required files such as `style.css` and `functions.php`. I learned that the `style.css` file not only stores custom CSS but also contains important information that links the child theme to the parent theme. The `functions.php` file is used to properly load both parent and child styles, which is necessary for the site to work correctly.

I also applied multiple visible customisations using CSS, including:
- Changing the overall background colour of the website  
- Modifying the navigation menu style and hover effects  
- Redesigning buttons with different colours and rounded edges  
- Updating the footer with a gradient background and improved spacing  
- Adding shadows and rounded corners to images and sections  
- Adjusting heading styles such as font size, colour, and spacing  

These changes helped me understand how CSS directly controls the appearance of different elements on a website. I was able to see how even small changes can improve the overall design and user experience.

Using the browser inspect tool was especially helpful during this process. It allowed me to test changes in real time before adding them to the code, which made it easier to experiment and understand how styles are applied.
